Voice consistency rule

Every dialogue line tagged james: or mara: reuses the same Comfy ElevenLabs voice workflow and settings. The user can rerun one line without changing the character voice profile.

Lock the environments.

Create the screenplay script.

The app can copy a ready LLM brief using story, cast, locations, voice slugs, and the required screenplay format. The returned script becomes the source for dialogue, shots, voices, and timeline assembly.

Required screenplay format

Scene 1: Motel lobby at night Location: motel_lobby Time: 0:00 Action: James counts cash at the front desk while rain taps the glass. Dialogue: james: "You need a room?" mara: "I already have one." Shot note: Start wide, then move into close coverage as the unease builds. Scene 2: Hallway outside Room 12 Location: hallway_12 Time: 0:42 Action: James follows Mara down the corridor. The lights buzz one by one.

Text to Speech

Default path. Best for one character line at a time and clean timeline placement.

Text to Dialogue

Optional whole exchange mode for two-character scenes when speed matters more than line-level control.

Speech to Speech

User records temp acting, then Comfy ElevenLabs converts it into the selected character voice.

Working principle

A short film is a script plus continuity. Characters keep their visual refs and voice profiles. Locations keep their reference slugs. Shots only describe coverage.

Open UX questions

Should voice generation happen line-by-line only, or should Text to Dialogue be offered for complete scenes? Should keyframes wait until all dialogue audio is ready?
